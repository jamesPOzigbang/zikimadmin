# 리포트 발급 이력 페이지 PRD

## 📄 페이지 정보

- **파일명**: `index.html`
- **페이지명**: 리포트 발급 이력
- **URL 경로**: `/index.html`

---

## 1. 개요

### 1.1 페이지 목적

리포트 발급 이력 페이지는 지킴진단 시스템에서 발급된 모든 리포트의 목록을 조회하고 관리할 수 있는 페이지입니다. 관리자는 이 페이지를 통해 리포트를 검색하고, 필터링하며, 상세 정보를 확인할 수 있습니다.

### 1.2 주요 기능

- 리포트 목록 조회
- 리포트 번호 검색
- 날짜 범위 필터링
- 리포트 상세 페이지로 이동
- 엑셀 다운로드

### 1.3 사용자 시나리오

관리자가 시스템에 로그인한 후, 리포트 발급 이력 메뉴를 클릭하여 이 페이지에 접근합니다. 리포트 목록을 확인하고, 필요에 따라 검색이나 필터링을 통해 특정 리포트를 찾아 상세 정보를 확인할 수 있습니다.

---

## 2. 정책

### 2.1 접근 권한

- **관리자 권한**: 리포트 발급 이력 조회 및 관리 권한이 있는 관리자만 접근 가능

### 2.2 데이터 표시 정책

- **페이지네이션**: 한 페이지에 표시되는 리포트 수는 기본 15개 (변경 가능)
- **정렬**: 기본 정렬은 리포트 발급 일시 기준 내림차순 (최신순)
- **날짜 형식**: 리포트 일시는 `YYYY-MM-DD HH:mm` 형식으로 표시
- **주소 형식**: 부동산 주소는 도로명 주소로 표시

### 2.3 검색 및 필터링 정책

- **검색 범위**: 리포트 번호로만 검색 가능
- **날짜 범위**: 시작일과 종료일을 지정하여 해당 기간 내 발급된 리포트만 조회
- **필터 적용**: 검색어와 날짜 범위 필터는 동시에 적용 가능

### 2.4 데이터 내보내기 정책

- **엑셀 다운로드**: 현재 화면에 표시된 리포트 목록을 엑셀 파일로 다운로드 가능
- **파일 형식**: `.xlsx` 형식
- **데이터 범위**: 현재 필터링된 결과만 다운로드

## 3. 기능

### 3.1 리포트 목록 조회

#### 3.1.1 기능 설명
페이지 로드 시 리포트 발급 이력 목록을 테이블 형태로 표시합니다.

#### 3.1.2 표시 항목
- **리포트 번호**: 리포트 고유 식별자 (예: `RPT-2023-1000`)
- **리포트 일시**: 리포트 발급 일시
- **주소**: 부동산 도로명 주소
- **주용도**: 건물 용도 (아파트, 빌라, 오피스텔 등)
- **리포트 목적**: 리포트 발급 목적 (매매, 전세 등)
- **결제 방식**: 결제 수단 (카드결제 등)
- **진행현황**: 리포트 처리 상태 (완료, 진행중, 취소)

#### 3.1.3 상태 표시
- **완료**: 초록색 배지
- **진행중**: 노란색 배지
- **취소**: 빨간색 배지

### 3.2 검색 기능

#### 3.2.1 리포트 번호 검색
- **입력 필드**: 리포트 번호 입력 필드 제공
- **검색 방식**: 리포트 번호의 일부 또는 전체 입력으로 검색
- **실시간 검색**: 입력 즉시 검색 결과 반영 (또는 검색 버튼 클릭 시)

### 3.3 날짜 범위 필터링

#### 3.3.1 날짜 선택
- **시작일**: 시작 날짜 선택 (date input)
- **종료일**: 종료 날짜 선택 (date input)
- **기본값**: 최근 1개월 또는 전체 기간

#### 3.3.2 필터 적용
- 날짜 범위 선택 후 자동으로 필터링 적용 또는 적용 버튼 클릭

### 3.4 리포트 상세 조회

#### 3.4.1 상세 페이지 이동
- 리포트 목록의 행을 클릭하면 해당 리포트의 상세 페이지로 이동
- URL 파라미터로 리포트 ID 전달: `report-detail.html?id={리포트번호}`

### 3.5 엑셀 다운로드

#### 3.5.1 다운로드 기능
- **버튼 위치**: 테이블 상단 액션 영역
- **다운로드 범위**: 현재 화면에 표시된 리포트 목록 (필터링된 결과 포함)
- **파일명 형식**: `리포트_발급이력_YYYYMMDD.xlsx`

### 3.6 페이지네이션

#### 3.6.1 페이지 이동
- **페이지 정보**: 총 항목 수 표시 (예: "Total 1,234 items")
- **페이지 버튼**: 이전/다음 페이지 이동 버튼
- **페이지 번호**: 현재 페이지 하이라이트

## 4. 유저스토리

### 4.1 리포트 목록 조회

**As a** 관리자  
**I want to** 리포트 발급 이력 목록을 조회할 수 있어야  
**So that** 발급된 리포트의 전체 현황을 파악할 수 있다

**Acceptance Criteria:**
- 페이지 로드 시 최신 리포트 목록이 표시된다
- 리포트 번호, 일시, 주소, 주용도, 목적, 결제 방식, 진행현황이 표시된다
- 진행현황은 색상 배지로 구분되어 표시된다

### 4.2 리포트 검색

**As a** 관리자  
**I want to** 리포트 번호로 검색할 수 있어야  
**So that** 특정 리포트를 빠르게 찾을 수 있다

**Acceptance Criteria:**
- 리포트 번호 입력 필드가 제공된다
- 리포트 번호의 일부만 입력해도 검색이 가능하다
- 검색 결과가 즉시 테이블에 반영된다

### 4.3 날짜 범위 필터링

**As a** 관리자  
**I want to** 특정 기간 내 발급된 리포트만 조회할 수 있어야  
**So that** 기간별 리포트 발급 현황을 분석할 수 있다

**Acceptance Criteria:**
- 시작일과 종료일을 선택할 수 있다
- 선택한 날짜 범위 내의 리포트만 표시된다
- 날짜 범위는 검색어와 함께 적용 가능하다

### 4.4 리포트 상세 조회

**As a** 관리자  
**I want to** 리포트 목록에서 리포트를 클릭하여 상세 정보를 확인할 수 있어야  
**So that** 리포트의 상세 내용을 확인할 수 있다

**Acceptance Criteria:**
- 리포트 목록의 행을 클릭하면 상세 페이지로 이동한다
- 상세 페이지에는 리포트의 모든 정보가 표시된다

### 4.5 엑셀 다운로드

**As a** 관리자  
**I want to** 리포트 목록을 엑셀 파일로 다운로드할 수 있어야  
**So that** 리포트 데이터를 외부에서 분석하거나 보관할 수 있다

**Acceptance Criteria:**
- 엑셀 다운로드 버튼이 제공된다
- 현재 화면에 표시된 리포트 목록이 엑셀 파일로 다운로드된다
- 엑셀 파일에는 모든 리포트 정보가 포함된다

### 4.6 페이지네이션

**As a** 관리자  
**I want to** 여러 페이지에 걸쳐 리포트 목록을 탐색할 수 있어야  
**So that** 대량의 리포트 데이터를 효율적으로 조회할 수 있다

**Acceptance Criteria:**
- 한 페이지에 표시되는 리포트 수가 제한되어 있다
- 페이지 이동 버튼이 제공된다
- 총 항목 수가 표시된다
