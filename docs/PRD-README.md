# 지킴진단 Admin PRD (Product Requirements Document)

## 📋 문서 개요

이 문서는 **지킴진단 관리자 시스템**의 제품 요구사항 명세서(PRD)입니다. 관리자가 리포트 발급 이력을 조회하고, 리포트에 표시되는 컨텐츠를 관리할 수 있는 시스템의 전체적인 개요와 각 페이지별 상세 요구사항을 포함합니다.

## 🎯 프로젝트 목적

지킴진단 관리자 시스템은 부동산 거래 시 필요한 진단 리포트를 관리하고, 리포트에 표시되는 컨텐츠를 효율적으로 관리하기 위한 관리자 도구입니다.

### 주요 목표
- 리포트 발급 이력의 체계적인 관리
- 리포트 컨텐츠의 중앙 집중식 관리
- 리포트 상세 정보의 명확한 조회 및 분석

## 📄 페이지 구조

이 시스템은 총 3개의 주요 페이지로 구성됩니다:

1. **[리포트 발급 이력](./리포트-발급-이력.md)** (`index.html`)
   - 리포트 발급 이력 목록 조회
   - 리포트 검색 및 필터링
   - 리포트 상세 페이지로 이동

2. **[컨텐츠 관리](./컨텐츠-관리.md)** (`content-management-standalone.html`)
   - 리포트에 표시되는 안내 문구 관리
   - 매매/전세/월세, 집합건물/일반건물별 컨텐츠 관리
   - 체크리스트 항목별 상태(양호/확인 필요)별 컨텐츠 관리

3. **[리포트 상세](./리포트-상세.md)** (`report-detail.html`)
   - 리포트의 상세 정보 조회
   - 매물 요약 정보
   - 진단 항목 상세
   - 범죄/치안 정보
   - 생활/편의 정보

## 👥 대상 사용자

- **관리자**: 리포트 발급 이력을 관리하고 컨텐츠를 수정하는 시스템 관리자
