# 컨텐츠 관리 페이지 PRD

## 📄 페이지 정보

- **파일명**: `content-management-standalone.html`
- **페이지명**: 리포트 컨텐츠 관리
- **URL 경로**: `/content-management-standalone.html`

---

## 1. 개요

### 1.1 페이지 목적

컨텐츠 관리 페이지는 리포트에 표시되는 주요 안내 문구를 관리하는 페이지입니다. 관리자는 이 페이지를 통해 리포트의 각 체크리스트 항목에 대한 안내, 설명, 특약 내용을 수정하고 관리할 수 있습니다.

### 1.2 주요 기능

- 리포트 타입별 컨텐츠 관리 (매매/전세/월세, 집합건물/일반건물)
- 체크리스트 항목별 컨텐츠 관리
- 상태별(양호/확인 필요) 컨텐츠 관리
- 컨텐츠 타입별(안내/설명/특약) 관리
- 동적 필드 추가 및 삭제
- 컨텐츠 저장 및 수정 이력 관리

### 1.3 사용자 시나리오

관리자가 리포트에 표시되는 안내 문구를 수정해야 할 때, 컨텐츠 관리 메뉴를 통해 이 페이지에 접근합니다. 리포트 타입(매매/전세/월세, 집합건물/일반건물)을 선택하고, 각 체크리스트 항목의 상태(양호/확인 필요)에 따라 표시될 컨텐츠를 수정합니다.

## 2. 정책

### 2.1 접근 권한

- **관리자 권한**: 컨텐츠 수정 권한이 있는 관리자만 접근 가능
- **인증**: 관리자 인증이 완료된 사용자만 접근 가능

### 2.2 컨텐츠 분류 정책

#### 2.2.1 리포트 타입 분류
리포트는 다음 6가지 타입으로 분류됩니다:
- 매매-집합건물
- 매매-일반건물
- 전세-집합건물
- 전세-일반건물
- 월세-집합건물
- 월세-일반건물

#### 2.2.2 상태 분류
각 체크리스트 항목은 다음 2가지 상태로 분류됩니다:
- **양호**: 특이사항이 없는 경우
- **확인 필요**: 추가 확인이 필요한 경우

#### 2.2.3 컨텐츠 타입 분류
각 상태별로 다음 3가지 컨텐츠 타입을 관리합니다:
- **안내**: 결과에 대한 안내 설명
- **설명**: 항목에 대한 정의 및 설명
- **특약**: 특약 사항 내용

### 2.3 컨텐츠 수정 정책

- **저장 시점**: 각 컨텐츠 타입별로 개별 저장 가능
- **수정 이력**: 최종 수정 일시가 각 항목에 표시됨
- **동시 수정**: 여러 항목을 동시에 수정 가능하나, 저장은 개별적으로 수행

### 2.4 특수 항목 정책

- **토지별도등기**: 이 항목에만 토지 유형 드롭다운이 제공됨
  - 구분지상권
  - 근저당권
  - 기타

### 2.5 동적 필드 관리 정책

- **필드 추가**: 각 컨텐츠 타입별로 여러 개의 필드를 추가할 수 있음
- **필드 삭제**: 추가된 필드는 삭제 가능 (최소 1개는 유지)
- **필드 구조**: 각 필드는 제목과 내용으로 구성됨

## 3. 기능

### 3.1 리포트 타입 선택

#### 3.1.1 탭 네비게이션
- **탭 구성**: 6개의 리포트 타입 탭 제공
- **탭 전환**: 탭 클릭 시 해당 타입의 컨텐츠로 전환
- **활성 탭 표시**: 현재 선택된 탭은 강조 표시

### 3.2 체크리스트 항목 관리

#### 3.2.1 체크리스트 항목 목록
총 21개의 체크리스트 항목이 그리드 형태로 표시됩니다:
- 위반건축물
- 대지권
- 토지별도등기
- 가등기
- 압류/가압류
- 처분금지가처분
- 신탁
- 경매개시결정
- 근저당권
- 임차권등기명령
- 전세권
- 민간임대주택등록 여부
- 등기사항 변경예정
- 건물,토지 소유자 일치
- 임대사업자 등록
- 보증금미반환 이력
- 기존채무금액
- 여유금액
- 보증보험 예비심사
- 보증금 대출 예비 심사
- 최우선 변제권

#### 3.2.2 항목 카드 구조
각 항목은 카드 형태로 표시되며 다음 정보를 포함합니다:
- 항목명
- 최종 수정 일시
- 상태 탭 (양호/확인 필요)
- 컨텐츠 입력 영역

### 3.3 상태별 컨텐츠 관리

#### 3.3.1 상태 탭 전환
- **양호 탭**: 양호 상태일 때 표시될 컨텐츠 관리
- **확인 필요 탭**: 확인 필요 상태일 때 표시될 컨텐츠 관리
- **탭 전환**: 상태 탭 클릭 시 해당 상태의 컨텐츠 입력 영역으로 전환

### 3.4 컨텐츠 타입별 관리

#### 3.4.1 라디오 버튼 선택
각 상태별로 다음 3가지 컨텐츠 타입을 선택할 수 있습니다:
- **안내**: 사용자 가이드 내용
- **설명**: 항목 정의 및 설명
- **특약**: 특약 사항

#### 3.4.2 컨텐츠 입력 영역 전환
- 라디오 버튼 선택 시 해당 타입의 입력 영역이 표시됨
- 한 번에 하나의 타입만 활성화됨

### 3.5 동적 필드 관리

#### 3.5.1 필드 추가
- **추가 버튼**: 각 컨텐츠 타입별로 "추가하기" 버튼 제공
- **필드 구조**: 제목 입력 필드와 내용 입력 필드로 구성
- **다중 필드**: 여러 개의 필드를 추가하여 리스트 형태로 관리 가능

#### 3.5.2 필드 삭제
- **삭제 버튼**: 추가된 필드에 삭제 버튼 제공
- **최소 개수**: 최소 1개의 필드는 유지되어야 함

### 3.6 특수 항목 처리

#### 3.6.1 토지별도등기 항목
- **토지 유형 드롭다운**: 이 항목에만 토지 유형 선택 드롭다운 제공
- **선택 옵션**: 구분지상권, 근저당권, 기타
- **동적 업데이트**: 토지 유형 선택에 따라 컨텐츠가 달라질 수 있음

### 3.7 컨텐츠 저장

#### 3.7.1 저장 기능
- **저장 버튼**: 각 컨텐츠 타입별로 "저장하기" 버튼 제공
- **저장 범위**: 현재 활성화된 컨텐츠 타입의 내용만 저장
- **저장 확인**: 저장 완료 시 알림 메시지 표시

#### 3.7.2 수정 이력 관리
- **최종 수정 일시**: 저장 시 자동으로 최종 수정 일시 업데이트
- **일시 표시 형식**: `YYYY.MM.DD HH:mm` 형식

### 3.8 UI/UX 기능

#### 3.8.1 애니메이션
- 탭 전환 시 페이드 인 애니메이션
- 필드 추가 시 부드러운 등장 애니메이션

#### 3.8.2 반응형 디자인
- 그리드 레이아웃으로 다양한 화면 크기에 대응
- 최소 너비 320px의 카드로 구성

## 4. 유저스토리

### 4.1 리포트 타입별 컨텐츠 관리

**As a** 관리자  
**I want to** 리포트 타입(매매/전세/월세, 집합건물/일반건물)별로 컨텐츠를 관리할 수 있어야  
**So that** 각 리포트 타입에 맞는 맞춤형 안내 문구를 제공할 수 있다

**Acceptance Criteria:**
- 6개의 리포트 타입 탭이 제공된다
- 탭 클릭 시 해당 타입의 컨텐츠로 전환된다
- 각 타입별로 독립적으로 컨텐츠를 관리할 수 있다

### 4.2 상태별 컨텐츠 관리

**As a** 관리자  
**I want to** 체크리스트 항목의 상태(양호/확인 필요)별로 다른 컨텐츠를 관리할 수 있어야  
**So that** 상태에 따라 적절한 안내 문구를 표시할 수 있다

**Acceptance Criteria:**
- 각 항목에 양호/확인 필요 탭이 제공된다
- 탭 전환 시 해당 상태의 컨텐츠 입력 영역이 표시된다
- 각 상태별로 독립적으로 컨텐츠를 관리할 수 있다

### 4.3 컨텐츠 타입별 관리

**As a** 관리자  
**I want to** 안내, 설명, 특약 등 컨텐츠 타입별로 내용을 관리할 수 있어야  
**So that** 리포트에 표시될 다양한 형태의 컨텐츠를 체계적으로 관리할 수 있다

**Acceptance Criteria:**
- 각 상태별로 안내/설명/특약 라디오 버튼이 제공된다
- 라디오 버튼 선택 시 해당 타입의 입력 영역이 표시된다
- 각 타입별로 독립적으로 컨텐츠를 저장할 수 있다

### 4.4 동적 필드 추가

**As a** 관리자  
**I want to** 각 컨텐츠 타입에 여러 개의 필드를 추가할 수 있어야  
**So that** 복잡한 안내 내용을 구조화하여 관리할 수 있다

**Acceptance Criteria:**
- "추가하기" 버튼이 제공된다
- 버튼 클릭 시 새로운 필드(제목 + 내용)가 추가된다
- 추가된 필드는 삭제할 수 있다

### 4.5 컨텐츠 저장

**As a** 관리자  
**I want to** 수정한 컨텐츠를 저장할 수 있어야  
**So that** 변경 사항이 리포트에 반영될 수 있다

**Acceptance Criteria:**
- 각 컨텐츠 타입별로 "저장하기" 버튼이 제공된다
- 저장 완료 시 알림 메시지가 표시된다
- 저장 시 최종 수정 일시가 업데이트된다

### 4.6 수정 이력 확인

**As a** 관리자  
**I want to** 각 항목의 최종 수정 일시를 확인할 수 있어야  
**So that** 컨텐츠의 최신성과 관리 이력을 파악할 수 있다

**Acceptance Criteria:**
- 각 항목 카드에 최종 수정 일시가 표시된다
- 저장 시 자동으로 수정 일시가 업데이트된다
- 일시는 읽기 쉬운 형식으로 표시된다

### 4.7 특수 항목 처리

**As a** 관리자  
**I want to** 토지별도등기 항목에 토지 유형을 선택할 수 있어야  
**So that** 토지 유형에 따라 적절한 컨텐츠를 제공할 수 있다

**Acceptance Criteria:**
- 토지별도등기 항목에만 토지 유형 드롭다운이 표시된다
- 구분지상권, 근저당권, 기타 옵션을 선택할 수 있다
- 선택한 토지 유형에 따라 컨텐츠가 달라질 수 있다
