# 지킴진단 관리자 시스템 PRD

> 💡 **Product Requirements Document**
> 부동산 리포트 발급 서비스의 관리자용 컨텐츠 관리 시스템

---

# 1. 개요

## 1.1 제품명

**지킴진단 관리자 시스템 (Jikim Admin System)**

## 1.2 목적

부동산 리포트 발급 서비스의 관리자가 리포트에 표시되는 컨텐츠를 효율적으로 관리할 수 있는 웹 기반 관리 시스템

## 1.3 대상 사용자

- 지킴진단 서비스 관리자
- 컨텐츠 운영 담당자

---

# 2. 핵심 기능 요구사항

## 2.1 리포트 발급 이력 관리

리포트 발급 내역을 조회하고 관리하는 기능

**주요 기능**

- 발급 이력 테이블 뷰
- 검색 및 필터링
- 날짜 범위 조회
- 상세 정보 모달

## 2.2 컨텐츠 관리 시스템

### 2.2.1 거래 유형별 탭 구조

6가지 거래 유형에 대한 독립적인 컨텐츠 관리

**거래 유형**

1. 매매 - 집합건물
2. 매매 - 일반건물
3. 전세 - 집합건물
4. 전세 - 일반건물
5. 월세 - 집합건물
6. 월세 - 일반건물

### 2.2.2 체크리스트 항목

각 거래 유형마다 동일한 21개 체크리스트 항목 관리

<details>
<summary><strong>체크리스트 항목 전체 보기 (21개)</strong></summary>

1. 위반건축물
2. 대지권
3. 토지별도등기
4. 가등기
5. 압류/가압류
6. 처분금지가처분
7. 신탁
8. 경매개시결정
9. 근저당권
10. 임차권등기명령
11. 전세권
12. 민간임대주택등록 여부
13. 등기사항 변경예정
14. 건물,토지 소유자 일치
15. 임대사업자 등록
16. 보증금미반환 이력
17. 기존채무금액
18. 여유금액
19. 보증보험 예비심사
20. 보증금 대출 예비 심사
21. 최우선 변제권

</details>

### 2.2.3 3단계 상태 분류 시스템

각 체크리스트 항목은 3가지 상태로 분류

| 상태 | 설명 | 색상 |
| --- | --- | --- |
| 🟢 안전 (Safe) | 문제없는 상태 | 녹색 계열 |
| 🟡 주의 (Caution) | 주의가 필요한 상태 | 노란색 계열 |
| 🔴 경고 (Warning) | 위험한 상태 | 빨간색 계열 |

### 2.2.4 컨텐츠 타입별 입력 시스템

각 상태별로 3가지 컨텐츠 타입 중 하나를 선택하여 입력

| 타입 | 설명 |
| --- | --- |
| 📖 정의 (Definition) | 해당 항목에 대한 정의 설명 |
| 📝 안내 (Guide) | 사용자에게 제공할 가이드 |
| 📋 특약 (Special Terms) | 특약 사항 |

### 2.2.5 동적 입력 필드 관리

각 컨텐츠 타입별로 여러 개의 입력 필드를 추가/삭제 가능

**입력 필드 구성**

- **제목 입력**: 2줄 텍스트 영역 (textarea)
- **본문 입력**: 3줄 이상 텍스트 영역 (textarea, 크기 조절 가능)
- **추가 버튼**: 새로운 입력 필드 추가
- **삭제 버튼**: 기존 입력 필드 제거 (첫 번째 필드 제외)

# 3. 사용자 인터페이스 요구사항

## 3.1 레이아웃 구조

### 3.1.1 사이드바 네비게이션

- **로고 영역**: 지킴진단 Admin 브랜딩
- **메뉴 항목**:
  - 📊 리포트 발급 이력
  - ⚙️ 컨텐츠 관리

### 3.1.2 상단 헤더

- **브레드크럼**: 현재 위치 표시
- **사용자 프로필**: 관리자 정보 표시

### 3.1.3 메인 컨텐츠 영역

- 페이지 제목 및 설명
- 주요 액션 버튼 (저장 등)
- 컨텐츠 영역

## 3.2 컨텐츠 관리 페이지 UI

### 3.2.1 탭 네비게이션

- 6개의 거래 유형 탭
- 활성 탭 시각적 강조
- 부드러운 전환 애니메이션

### 3.2.2 체크리스트 그리드

- 반응형 그리드 레이아웃 (최소 320px 컬럼)
- 카드 형태의 각 체크리스트 항목
- 호버 시 시각적 피드백

### 3.2.3 상태 탭 UI

- 3개의 상태 토글 버튼 (안전/주의/경고)
- 상태별 색상 구분

### 3.2.4 컨텐츠 입력 영역

- 라디오 버튼으로 컨텐츠 타입 선택
- 동적으로 추가/제거 가능한 입력 필드
- 제목과 본문 구분된 입력 영역
- 플레이스홀더로 입력 가이드 제공

# 6. 사용자 시나리오

## 6.1 컨텐츠 수정 시나리오

1. 관리자가 "컨텐츠 관리" 메뉴 클릭
2. 원하는 거래 유형 탭 선택 (예: 매매-집합건물)
3. 수정할 체크리스트 항목 찾기 (예: 위반건축물)
4. 상태 탭 선택 (안전/주의/경고)
5. 컨텐츠 타입 선택 (정의/안내/특약)
6. 제목과 본문 입력
7. 필요시 "+ 추가하기" 버튼으로 추가 입력 필드 생성
8. 모든 수정 완료 후 "변경사항 저장" 버튼 클릭

## 6.2 다중 컨텐츠 입력 시나리오

1. 특정 항목의 "안전" 상태에서 "안내" 타입 선택
2. 첫 번째 안내 문구 입력
3. "+ 추가하기" 클릭하여 두 번째 입력 필드 생성
4. 두 번째 안내 문구 입력
5. 필요시 추가 입력 필드 계속 생성
6. 불필요한 필드는 "삭제" 버튼으로 제거
